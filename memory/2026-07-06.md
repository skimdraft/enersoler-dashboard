### 7. Objectif journalier
- Formule corrigée : `capacity_kwp × 3.84` (pas 4.5)
- Paea : 50.5 × 3.84 = 194 kWh, Temana : 58 × 3.84 = 223 kWh

### 8. Historique persistant
- Problème : `generateFakeHistory()` utilisait Math.random() → valeurs changeaient à chaque refresh
- Solution : `generate-history.js` + `history-daily.json`
  - Seed déterministe (mulberry32) stocké dans `.history-seed`
  - 365 jours de données quotidiennes (pas de points 5-min)
  - Saisonnalité : Jan-Mar = 45% (pluies), Apr-Sep = 70%, Oct-Dec = 95%
  - Format : `[{date, kwh, paea_kwh?, temana_kwh?}]`
  - Merge automatique avec vraies données dans `build.js`
  - ~50 KB (364 entrées) vs ~8 MB pour des points 5-min
- Données stockées dans :
  - `isolar-data.json` — snapshot live (écrasé toutes les 60s)
  - `history.json` — points 5-min du jour
  - `history-daily.json` — historique PERSISTANT (364 jours)
  - `.history-seed` — graine déterministe

### 9. API historique — ÉCHEC
- Testé 8+ endpoints historiques iSolarCloud → tous retournent E900 "Unauthorized"
- Notre app OAuth2 n'a que les permissions temps réel
- L'interface web (web3.isolarcloud.com.hk) utilise une API interne différente
- Options : modifier l'enregistrement de l'app sur la plateforme dev iSolarCloud, ou densifier le polling

### 10. Coût tokens
- Le cron local n'utilise AUCUN token LLM (juste Node.js)
- Seules les conversations avec le LLM consomment des tokens
