# 2026-07-01 — Memory Log

## Solar Dashboard — Major Data Correction

### The Problem
Dashboard was showing absurd values: 394 kW live power on 108 kWp installed capacity, nonsense daily production. User called it out as "gros bullshit."

### Root Cause Analysis
- **p1** registers are NOT instantaneous active power in Watts — they accumulate in unknown units and diverge from DC/apparent power throughout the day
- **p3** registers are NOT daily energy — barely changed (+6 Wh in 9 hours while actual production was 150+ kWh)
- **p88** registers are NOT monthly/yearly energy — they increment identically to p2 (just with different baselines)

### Verified Correct Register Map (SG50CX-P2 inverter)
| Point | Meaning | Verification |
|-------|---------|-------------|
| p2 | Total lifetime energy (Wh) | ✅ Cumulates correctly |
| p4 | Inverter temperature (°C) | ✅ Reasonable values |
| p14 | Total DC power (W) | ✅ Consistent with p24 |
| p24 | Apparent AC power (VA) | ✅ Consistent with p14 |
| p26 | Power factor (cos φ) | ✅ |
| p86 | Load ratio = p14/Capacity | ✅ Verified: 0.266 → 13434/50460 = 0.266 |
| p1, p3, p88 | UNRELIABLE — removed from dashboard | ❌ |

### Daily Energy Solution
- iSolarCloud OAuth2 apps CANNOT access production statistics endpoints (E900 unauthorized)
- Only solution: daily energy = p2(current) - p2(midnight baseline)
- Baseline stored in `dashboard/daily-baseline.json`
- First baseline captured at 16:01, so daily energy pending until midnight reset
- After midnight: daily energy will be accurate and reset daily
